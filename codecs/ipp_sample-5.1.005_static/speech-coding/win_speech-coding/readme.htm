Overview

This package contains the speech codec sample (./application/usc_speech_codec) and the echo cancellation sample (./application/usc_ec).

The speech codec sample supports encoding and decoding compliant to the ITU-T G.711/I/II, G.722, G.722.1, G.723.1/A, G.726/A, G.728G//I/H, G.729/A/B/D/E/I/C/C+ recommendations, ETSI GSM 06.90-06.94, 06.10-06.12, 06.31-06.32 recommendations.

The echo cancellation sample complies to the ITU-T G.168-2000 and G.167 recommendations. 

The samples have been implemented using the Intel® Integrated Performance Primitives (Intel® IPP) Speech Coding functions. They are command line based applications which process data given by one or more input PCM files and store the result into output bitstreams or PCM files. Both narrow band (8000 Hz)  and wide band (16000 KHz) are supported.  

For more information refer to the specific sample readme.htm file.

System Requirements

Recommended hardware:  

  • A system based on the Intel® Pentium® 4 processor with 128 megabytes of RAM, or better.
  • A system based on a Intel® Itanium® architecture with 512 megabytes of RAM, or better
  • A system based on Intel® Pentium® D processor with Intel® Extended Memory 64 Technology (Intel® EM64T)
  • A system based on the Intel® IXP425 processor.

Hardware requirements:

  • A system based on a Intel® Pentium® processor, Intel® Xeon® processor, or subsequent IA-32 processor
  • A system based on a Intel® Itanium®  2 processor
  • A system based on a Intel® Xeon® processor with Intel® Extended Memory 64 Technology (Intel® EM64T)
  • A system based on the Intel® IXP4xx processor.

Software requirements: 

  • Intel® IPP 5.1 for Windows*, or later
  • Intel® C++ Compiler 8.1 for Windows*, or later
  • Microsoft* Visual C++* 6.0 or Microsoft* Visual C++* .NET*
  • Microsoft* Windows* 2003 Server
  • Microsoft* Windows* 2000, or Windows* XP
  • For an Intel® Xeon® processor with Intel® EM64T: Microsoft* Platform SDK for Windows* Server 2003 SP1" for EM64T platform
  • For Intel XScale®:  Microsoft* eMbedded Visual C++* 4.0 SP4
  • When building for a Windows* CE 5.0 for x86, Standard SDK for Windows* CE 5.0 is required
  •  When building for a Windows* CE 5.0 for IXP 4XX product line processor, IXP 4XX product line SDK for Windows* CE 5.0 is required

Installation

Extract all files in w_ipp-sample-speech-coding_*.zip to a desired folder. Please make sure to preserve the directory structure. The files for the USC Speech Codec and Echo Cancellation sample will be found in the ./ipp_sample/speech-coding folder.

How to Build Source Code

  • Set up your build environment by creating an environment variable named IPPROOT that points to the root directory of your Intel® IPP installation. For example: IPPROOT=C:/Program Files/Intel/IPP/5.1/ia32/
    • To build for an IA-32 system: run build32.bat.
    • To build for an Intel® Xeon(TM) Processor with Intel® Extended Memory 64 Technology system: run buildem64t.bat.
    • To build for an Intel® Itanium® processor based system: run build64.bat.
    • To build for an Intel® IXP4xx processor: run buildwce420xsc.bat (XScale wce420) or buildwcexsc.bat (XScale wce500), buildwcex86.bat (IA32,WindowsCE)

Note for developers for WindowsCE 5.0.  Probably you'll need to set the SDKROOT environment variable manually, for example:  set SDKROOT=C:\Program Files\Windows CE Tools

The easiest way to build all application is to simply run build32.bat located under directory ./ipp_sample/speech-coding if the target system is based on Intel® Pentium® Processor system. The build*.bat will build the USC speech codec library and two samples, namely the unified echo canceller and the unified speech codec. If build is successful all relevant .obj files and .lib files are created and stored under directory ./ipp_sample/speech-coding/bin/ia32/obj/* and  ./ipp_sample/speech-coding/bin/ia32/lib/* respectively. The .exe files will be placed in ./ipp_sample/speech-coding/bin/ia32.

The USC library and samples can be built separately for desired platform using build*.bat batch files located in specific folders:

  1. ./ipp_sample/speech-coding/codec/build*.bat - to build the USC library  
  2. ./ipp_sample/speech-coding/application/usc_speech_codec/build*.bat - to build USC Speech Codec sample. The USC library must be built prior to the USC speech codec separate build.
  3.  ./ipp_sample/speech-coding/application/usc_ec/build*.bat - to build the Echo Canceller sample.

Note, with default settings it will build the sample with Intel C++ compiler. To change defaults see the Makefile for a brief description of the available variables and additional options.

Running the Software

Read the specific sample readme.htm file to learn how to build and run the sample (to locate see the table above).   

In order to run each executable created by this sample, the Intel® IPP DLLs must be on the system's path. DLLs are located in the bin subdirectory of the root directory of the Intel® IPP installation (IPPROOT). This can be done by setting the PATH environment variable manually or invoking the batch file: "%IPPROOT%/tools/env/ippenv.bat" from the Intel® IPP installation.
