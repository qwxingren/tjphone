Overview

The USC Echo Canceller sample is a command line based application that performs echo cancellation. The EC sample processes two input files and creates one output PCM file either narrow band 16bit 8000 Hz or wideband 16bit 16000 Hz. The receive-in and send-in samples are processed to perform echo path parameters estimation and estimated echo is subtracted from the send-in samples and "echo-free" PCM data is written to send-out output file. Both integer and floating-point implementation are supported.

The sample implements three algorithms:

1.      Fullband algorithm  - processing in time domain,

2.      Subband algorithm – processing in the frequency domain,

3.      Subband-Fast algorithm – fast processing in the frequency domain. The Subband-Fast algorithm introduces a small delay (about 20ms) into the output signal.

The echo canceller algorithm complies with ITU G.168-2002 specifications.

 

Running the Software

For information on building the sample, see the readme file in the ipp_sample\speech-coding directory.

In order to run the executable created by this sample, the Intel® IPP DLLs must be on the system's path. DLLs are located in the bin subdirectory of the root directory of the Intel® IPP installation (IPPROOT). This can be done by setting the PATH environment variable manually or invoking the batch file: "%IPPROOT%\tools\env\ippenv.bat" from the Intel® IPP installation.

To run the USC EC sample, type: 

usc_ec.exe [options] <r-inPCMFile> <s-inPCMFile> <s-outPCMFile>

Option list: 

     [-type <EC_FP|EC_INT>] – select implementation to execute: EC_INT - integer implementation, EC_FP - floating-point implementation.

     [-f  <frequency >] - sample frequency in KHz: 8 or 16. Default: 8.

     [-a <algorithm>] - specify an algorithm: fb (fullband), sb (subband) or sbf (subband-fast). Default: <algorithm> = sb.

     [-ad <mode>] - set adaptation mode: "full" or "lite". "full" is a default value. "lite" mode requires significantly low cpu resources but may show worse adaptation characteristics.

     [-hp <num>] - set the number of frames to be necessarily filtered after the howling has been detected. Default: sampling rate/frame size. Frame size is 64 - for sb mode, 8 for fb mode, 24 for sbf mode.

     [-ah <mode>] - set the anti-howling device on. Mode=[1,2] defines the howling detector device algorithm. 1-for spectra-based method, 2- for energy-based method.

     [-sh <num>] - set the frequency shift value in the anti-howling device. No shifting performed as default.

     [-l <num>] - echo tail length in ms in range [1, 200] for subband and subband-fast algorithms, [1, 16] for fullband algorithm. Default: <num> = 16.

     [-delay <val>]   Set the delay in receive-in PCM file, integer number of milliseconds. The leading end of receive-in PCM will be cut off prior to echo cancel. Default: no delay.

     [-s  <rep>] - repeater, how many times input files shall be processed without resetting EC,  default: one time. 

     [-c <csv filename>] - Print out performance info line to specified file (optional).

     [-o <logFile>] - write log information to the file  <logFile> .

     [-list] - print out the modes list.
     [-sys] - print out system information.
<r-inPCMFile> - input file contains receive-in 16-bit PCM signal.

<s-inPCMFile> - input file contains send-in 16-bit PCM signal.

<s-outPCMFile> - output file contains send-out 16-bit PCM signal.

 

Note:  if repeat is specified (-s option) the last result only will be written into the output file.
